# คู่มือบันทึกผล PM รอบ Q2 2026 (On-site)

> สำหรับ: ทีม IT Support  
> รอบ: Q2 2026 | ช่วงออกพื้นที่: ปลาย พ.ค. – กลาง ก.ค. 2569 | ปิดเอกสาร: 31 ก.ค. 2569  
> ลิงก์ระบบ: `https://pass-inventory.vercel.app/admin/`

---

## ขั้นตอนที่ 0 — Admin เตรียมระบบ (Nopporn ทำครั้งเดียวก่อนออกพื้นที่)

1. เข้า `https://pass-inventory.vercel.app/admin/`
2. เมนูซ้าย → **Pm rounds → Add PM round**
3. กรอก:

| ช่อง | ค่า |
|---|---|
| Year | 2026 |
| Quarter | Q2 |
| Field start date | 2026-05-25 |
| Field end date | 2026-07-15 |
| Document due date | 2026-07-31 |

4. กด **Save**
5. แจ้งทีมว่าระบบพร้อมแล้ว

---

## ขั้นตอนที่ 1 — เตรียมตัวก่อนออกพื้นที่

### 1.1 เตรียมข้อมูล
- เปิด `https://pass-inventory.vercel.app/devices/`
- กรอง **Plant ปัจจุบัน** = plant ที่จะไป
- จดหรือ screenshot รายการ S/N ทุกเครื่องที่ควรพบในวันนั้น

### 1.2 ทดสอบ login ล่วงหน้า
- เข้า `https://pass-inventory.vercel.app/admin/`
- ใส่ username / password ของตัวเอง
- ตรวจว่าเห็นเมนู **Pm snapshots** ในแถบซ้าย
- ถ้า login ไม่ได้ แจ้ง Nopporn ก่อนออกพื้นที่

### 1.3 สิ่งที่ต้องพกไป
- มือถือหรือ tablet ที่มีเน็ต (ใช้กรอกข้อมูลออนไลน์ระหว่าง on-site)
- รายการ S/N ที่พิมพ์หรือ screenshot ไว้จากข้อ 1.1

---

## ขั้นตอนที่ 2 — ถึง Plant แล้ว (ทำก่อนเริ่มตรวจเครื่อง)

1. Login ที่ `https://pass-inventory.vercel.app/admin/`
2. เมนูซ้าย → **Pm snapshots**
3. กด **Filter** ด้านขวา → เลือก Year=2026, Quarter=Q2, Plant=XXX
4. ดูว่ามีเครื่องไหนที่บันทึกไปแล้วบ้าง (กรณีที่เคยมาก่อน)
5. เปิดรายการ S/N ที่เตรียมไว้ — พร้อมเริ่มตรวจได้

---

## ขั้นตอนที่ 3 — ตรวจและบันทึกผลแต่ละเครื่อง

ทำซ้ำขั้นตอนนี้ทีละเครื่องจนครบ

### 3.1 ตรวจเครื่องจริง
- หยิบเครื่องขึ้นมาดู S/N (สติ๊กเกอร์หลังเครื่องหรือกล่อง)
- ตรวจสภาพ: เปิดติดไหม / จอแตกไหม / ชาร์จได้ไหม / มีรอยเสียหายไหม
- จดสภาพไว้ในใจหรือกระดาษก่อน

### 3.2 เปิดฟอร์มบันทึกในระบบ
- เมนูซ้าย → **Pm snapshots → Add PM snapshot**

### 3.3 กรอกฟอร์ม

**ช่อง Device**
- คลิกในช่อง Device แล้วพิมพ์ S/N ที่เห็นบนเครื่อง
- รอสักครู่ ระบบจะขึ้นรายการให้เลือก
- คลิกเลือก S/N ที่ตรงกัน
- ถ้าค้นไม่เจอ → ดูหัวข้อ "กรณีพิเศษ" ด้านล่าง

**ช่อง Pm round**
- เลือก **Q2 2026** เสมอ (ห้ามเลือกรอบอื่น)

**ช่อง Plant**
- เลือก plant ที่เครื่องอยู่ตอนนี้จริงๆ
- ถ้าเครื่องถูกยืมมาจาก plant อื่น → เลือก plant ที่เครื่องอยู่ตอนนี้ (ไม่ใช่ home plant)

**ช่อง Status**
- เลือกตามตารางนี้:

| สภาพที่เห็น | Status ที่เลือก |
|---|---|
| เครื่องปกติ ใช้งานได้ อยู่ที่ plant นี้ตามปกติ | `ใช้งานปกติ` |
| เพิ่งรับเครื่องใหม่จาก Enco ในรอบนี้ | `รับเครื่องใหม่` |
| เครื่องเสีย / ชำรุด / คืน Enco แล้ว | `เสื่อมสภาพ/คืน Enco` |
| เครื่องนี้ยืมมาจาก plant อื่น | `ยืมเข้ามาใช้` |
| เครื่อง plant นี้แต่ถูกยืมออกไปที่ plant อื่น | `ถูกยืมออกไป` |

**ช่อง Note**
- กรณีปกติ: ไม่ต้องกรอก
- กรณีมีปัญหา: กรอกอาการสั้นๆ เช่น "จอแตกมุมขวาล่าง", "เปิดไม่ติด ส่ง Enco 15/6/69"
- กรณียืม: กรอกว่า "ยืมจาก HDY" หรือ "ยืมให้ KBV"

### 3.4 กด Save
- ถ้าขึ้น "The PM snapshot was added successfully." → บันทึกสำเร็จ ไปเครื่องถัดไป
- ถ้าขึ้น error → ดูหัวข้อ "error ที่พบบ่อย" ด้านล่าง

## ขั้นตอนที่ 4 — นำ Tablet ใหม่ไปส่ง Plant

> ใช้เมื่อ: รอบ Q2 2026 มีการนำเครื่องใหม่จาก Enco ไปติดตั้งที่ plant

---

### กรณี A — เพิ่มเครื่องใหม่ (ไม่มีเครื่องเก่า)

**Admin (Nopporn) ทำก่อนออกพื้นที่:**

1. รับรายการ S/N เครื่องใหม่จากทีมหรือ Enco
2. เข้า Admin → **Devices → Add Device** แล้วกรอกทีละตัว:

| ช่อง | ค่า |
|---|---|
| Serial number | S/N ใหม่จาก Enco |
| Asset type | Tablet |
| Home plant | plant ที่จะรับเครื่อง |
| Received round | **Q2 2026** (สำคัญ — ใช้คำนวณวันประกัน) |
| Warranty months override | ใส่ถ้าประกันไม่ใช่ 12 เดือน ไม่ใส่ก็ได้ถ้าใช้ default |
| ช่องอื่น | ปล่อยว่าง |

3. กด **Save** → ทำซ้ำจนครบทุก S/N
4. แจ้งทีมว่าเครื่องพร้อมในระบบแล้ว

**Support ทำ on-site (หลังส่งเครื่องให้ plant แล้ว):**

1. เมนู Admin → **Pm snapshots → Add PM snapshot**
2. กรอก:

| ช่อง | ค่า |
|---|---|
| Device | S/N เครื่องใหม่ |
| Pm round | Q2 2026 |
| Plant | plant ที่รับเครื่อง |
| Status | `รับเครื่องใหม่` |
| Note | เช่น "รับเครื่องจาก Enco วันที่ XX/XX/XX" |

3. กด **Save**

---

### กรณี B — นำเครื่องใหม่มาแทนเครื่องเก่าที่เสีย

**Admin (Nopporn) ทำก่อนออกพื้นที่:**

1. **Add Device ใหม่** (ทำเหมือนกรณี A ข้างบน) แต่เพิ่มช่อง:

| ช่อง | ค่า |
|---|---|
| Replaced predecessor | เลือก S/N เครื่องเก่าที่ถูกแทน |

2. **Retire เครื่องเก่า** → ไปที่ Devices → ค้นหา S/N เก่า → แก้ไข:

| ช่อง | ค่า |
|---|---|
| Is retired | ✅ ติ๊กถูก |
| Retired round | Q2 2026 |

3. กด **Save**

**Support ทำ on-site:**

ต้องบันทึก PMSnapshot **2 rows** สำหรับรอบนี้:

**Row 1 — เครื่องเก่า (ที่คืน Enco):**

| ช่อง | ค่า |
|---|---|
| Device | S/N เครื่องเก่า |
| Pm round | Q2 2026 |
| Plant | plant นี้ |
| Status | `เสื่อมสภาพ/คืน Enco` |
| Note | เช่น "คืน Enco วันที่ XX/XX/XX แทนด้วย S/N ใหม่" |

**Row 2 — เครื่องใหม่ (ที่ส่งมาแทน):**

| ช่อง | ค่า |
|---|---|
| Device | S/N เครื่องใหม่ |
| Pm round | Q2 2026 |
| Plant | plant นี้ |
| Status | `รับเครื่องใหม่` |
| Note | เช่น "แทน S/N เก่า" |

## ขั้นตอนที่ 5 — กรณีพิเศษอื่นๆ

### เครื่องถูกยืมระหว่าง plant (ต้องบันทึก 2 rows)

ตัวอย่าง: เครื่อง RJB03F0074 บ้านอยู่ที่ CNX แต่ตอนนี้ถูกยืมไปอยู่ที่ KBV

**Row ที่ 1** (บันทึกตอนอยู่ที่ CNX หรือโทรถามได้):
- Device = RJB03F0074
- Plant = CNX
- Status = `ถูกยืมออกไป`
- Note = "ยืมให้ KBV"

**Row ที่ 2** (บันทึกตอนอยู่ที่ KBV):
- Device = RJB03F0074
- Plant = KBV
- Status = `ยืมเข้ามาใช้`
- Note = "ยืมจาก CNX"

### S/N ค้นหาไม่เจอในระบบ
1. ถ่ายรูป S/N จริงบนเครื่องไว้
2. แจ้ง Nopporn ทาง Line/โทร พร้อมส่งรูป
3. รอ Nopporn Add Device ในระบบก่อน
4. กลับมากรอก Snapshot ทีหลังได้ (ไม่ต้องรอหน้างาน)
5. **อย่าสร้างเครื่องใหม่เองในระบบ**

### S/N บนเครื่องไม่ตรงกับในระบบ
1. ถ่ายรูป S/N จริงไว้
2. ไม่ต้องบันทึกในรอบนี้ก่อน
3. แจ้ง Nopporn ตรวจสอบ — อาจเป็น S/N เก่าที่พิมพ์ผิดในระบบ

### เครื่องหาไม่เจอที่ plant
1. ถามเจ้าหน้าที่ plant ว่าเครื่องอยู่ที่ไหน
2. ถ้ายืมออกไป → กรอก status `ถูกยืมออกไป` + Note ว่ายืมไปที่ไหน
3. ถ้าหาไม่เจอจริงๆ → กรอก status `ใช้งานปกติ` + Note ว่า "ค้นหาไม่พบ ณ วันที่ XX/XX/XX" แล้วแจ้ง Nopporn

### บันทึกผิด (กด Save ไปแล้ว)
1. เมนูซ้าย → **Pm snapshots**
2. ค้นหา S/N เครื่องนั้นในช่อง Search
3. คลิกเข้าไปแก้ไข
4. แก้ช่องที่ผิด แล้วกด **Save**

### Error: "PM snapshot with this Device and Pm round already exists"
- แปลว่าเครื่องนี้บันทึก Q2 2026 ไปแล้ว (อาจมีคนกรอกไว้ก่อน)
- กด Cancel → ไปหา record เดิมแล้วแก้ไขแทน

## ขั้นตอนที่ 6 — ตรวจสอบก่อนออกจาก Plant

### วิธีตรวจแบบเร็ว (แนะนำ)
1. เปิด `https://pass-inventory.vercel.app/devices/`
2. กรอง **Plant ปัจจุบัน** = plant ที่กำลังอยู่
3. เปรียบเทียบกับรายการที่เตรียมไว้ตอนแรก
4. ถ้าจำนวนตรงกัน → พร้อมออกได้

### วิธีตรวจแบบละเอียด (ถ้ามีเวลา)
1. เมนู Admin → **Pm snapshots**
2. Filter: Year=2026, Quarter=Q2, Plant=XXX
3. ดูว่าครบทุก S/N ที่ควรมีหรือไม่

---

## ขั้นตอนที่ 7 — หลังกลับจาก on-site

1. แจ้ง Nopporn ว่าเสร็จ plant ไหนแล้ว
2. ถ้ามีเครื่องที่ค้างบันทึก (เพราะ S/N ไม่เจอ ฯลฯ) → แจ้งให้ Nopporn แก้ไขในระบบ แล้วกลับมากรอกให้ครบ
3. Nopporn ตรวจสอบภาพรวมผ่าน Admin ก่อนปิดเอกสาร 31 ก.ค. 2569
